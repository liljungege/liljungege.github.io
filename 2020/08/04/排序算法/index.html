排序算法

发表于 2020-08-04 分类于 数据结构

本文是对几种基础排序的实现以及算法分析，他们是冒泡排序、选择排序、插入排序、希尔排序、归并排序、快速排序、堆排序。

这里的排序指升序排序

模板函数

  • boolean less(Comparable v, Comparable w)：看v是否小于w，若小于则返回true，否则返回false。
  • void exch(Comparable[] a, int i, int j)：交换i和j位置上的元素。
  • void show(Comparable[] a)：打印数组
  • boolean isSorted(Comparable[] a)：看数组a是否有序，有序返回true，否则返回false。

冒泡排序

算法思想

比较相邻的两个元素，如果前一个数大于后一个数，则交换他们两个，每一对都要交换，交换完成后，最后的元素就是最大的数。

当输入数据已经是升序排列时，最快；当每一组数据都需要交换时，最慢。

对算法的一些改进

1、可以设置一个flag，判断上次循环是否进行过交换，当flag = true时，无需交换，跳出循环。

2、双向扫描，先从前往后遍历一次，再从后往前遍历一次。

算法分析

平均时间复杂度 O(n^2) ，最好情况 O(n) ，最坏情况O(n^2) ，是稳定排序。

选择排序

算法思想

首先找到数组中最小的那个元素，其次，将它和数组的第一个元素交换位置；再次，在剩下的元素中找到最小的元素，将它与第二个元素交换位置。如此反复，直到将整个数组排序。

在遍历时，记录最小值的坐标，如果他不是当前索引的那个值，就要进行交换，否则不交换。

算法分析

平均、最好、最坏时间复杂度 O(n^2)，是不稳定的。

插入排序

算法思想

与选择排序一样，当前索引左边的所有元素都是有序的，但他们的最终位置还不确定，为了给更小的元素腾出空间，他们可能会被移动。当索引到达数组右端时，数组就完成排序了。

和选择排序不同的是，插入排序所需时间取决于输入中元素的初始顺序。所以利用选择排序对一个很大且接近有序的数组排序要比随机顺序的数组或者逆序的数组要快得多。

对于1到N-1之间的每一个i，将a[i]与a[0]到a[i-1]中比它小的所有元素依次有序的交换。

算法分析

平均时间复杂度O(n^2)，最好 O(n)，最坏 O(n^2)，稳定。

希尔排序

是插入排序的改进版本，非稳定排序

算法思想

交换不相邻的元素以对数组的局部进行排序，并最终用插入排序将局部有序的数组排序。

使数组中任意间隔为h的元素都是有序的，这样的有序数组为h有序数组。

对于每个h，用插入排序将h个子数组独立地排序。但因为子数组是相互独立的，一个更简单的方法是在h-子数组中将每个元素交换到比它大的元素之前去。只需要在插入排序的代码中将移动元素的距离由1改为h即可。希尔排序的实现就转化为了一个类似于插入排序但使用不同增量的过程。

算法分析

平均时间复杂度O(n^2)，不稳定

快速排序

算法思想

挑选出一个基准，这个基准最好随机挑选，把所有比基准小的放在基准前面，所有比基准大的放在基准后面，对基准的左边和右边两个子集不断重复上述步骤，直到所有子集只剩下一个或零个元素。

算法分析

平均以及最好时间复杂度 O(nlogn)，最坏时间复杂度O(n^2)，空间O(logn)，不稳定

快排优点：内循环速度较快。

快排缺点：在切分不平衡时程序可能会非常低效。比如说，第一次从最小的元素切分，第二次从第二小的元素切分，以此类推。所以要在partition前随机化数组，或者随机选择a[l]。

普通快排 实现一

单路快排，从数组左端开始考虑起。

普通快排 实现二

先取a[l]作为切分元素，这里采用了随机取的方法。然后从数组的左端开始向右扫描直到找到一个大于等于它的元素，再从数组的右端开始向左扫描直到找到一个小于等于它的元素。这两个元素未排定，所以交换他们的位置。

算法改进

  1. 对于小数组，可以使用插入排序
  2. 三路快排
    • 针对于有大量重复元素的数组比较好用。
    • 将数组切分为三部分，分别对应小于、等于和大于切分元素的数组元素。

堆排序

算法思想

堆排序分为两个阶段：

  1. 在堆的构造阶段中，将原始数组重新组织安排进一个堆中。
  2. 在下沉排序阶段，从堆中按递减顺序取出所有元素并得到排序结果。

在这里使用最大堆，并重复删除最大元素。

堆的构造
