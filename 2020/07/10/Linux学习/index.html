基于Centos7

环境搭建

  1. 安装VMware虚拟机软件
  2. 在虚拟机上安装Centos7
    • 通过镜像进行安装 http://mirrors.aliyun.com/centos/7/isos/x86_64/
    • Linux磁盘分区的时候需要注意分区名即可 /boot 1024M swap 4000M / 剩下的
    • 点击屏幕进入虚拟机，CTRL+ALT退出虚拟机

  1. 设置网络(桥接模式)
    1. 利用dhclient自动分配网络内可用的ip地址（要是在root用户下）
    2. 利用ifconfig或ip addr查询当前网络状态
    3. 配置网卡地址 vim /etc/sysconfig/network-scripts/ifcfg-ens33
    4. 将其进行修改 网关要根据网络适配器中的值进行修改
    5. 重启网络服务 systemctl restart network.service
    6. 查看是否可以与外网以及宿主网络相连
  2. 利用Xshell连接Linux
  3. 利用Xftp上传文件
    • 解压上传的文件 tar -zxvf apache-tomcat-8.5.57.tar.gz

Linux系统

开机登陆

Linux开机后，会启动很多程序，在Linux中称为“守护进程”

关机

Linux大多用在服务器上，很少遇到关机的操作。

关机指令为：shutdown

系统目录结构

  1. 一切皆文件
  2. 根目录 /

登陆系统，输入ls /

Linux为树状目录结构（一切资源都挂载在 / 根节点下）

对这些目录的解释：

  • /bin：bin是Binary的缩写，这个目录存放着最经常使用的命令。
  • /boot：存放Linux核心文件。
  • /dev：device，存放Linux的外部设备。
  • /etc：用来存放所有的系统管理所需要的配置文件和子目录。如：网络配置，Redis配置等。
  • /home：用户的主目录，在Linux中，每个用户都有一个自己的目录，一般目录是以用户的账户命名的。
  • /lib：动态链接共享库，类似于Windows中的DLL文件
  • /lost+found：存放突然关机的文件
  • /opt：给主机额外安装软件所摆放的目录。比如说安装了ORACLE数据库，就可以放在这个目录下。默认是空的。
  • /root：该目录为系统管理员，也称作超级权限者的用户主目录。
  • /usr：用户的很多应用程序和文件都放在这个目录下。
  • /tmp：这个目录是用来存放一些临时文件的。用完即丢的文件，可以放在这个目录下，比如说：安装包。
  • /www：存放服务器网站相关的资源，环境，网站的项目。

常用的基本命令

目录管理

绝对路径、相对路径

绝对路径：C:\D\JAVA\VirtualMachine\CentOS7-03-GUI\vmware.log

相对路径：在CentOS7-03-GUI目录下，那vmware.log文件对应我们的相对配置就是 /vmware.log

  • cd: 切换目录命令
  • cd..：返回上一级目录
  • ./：代表当前目录

ls：列出目录

  • -a：all，查看所有的文件，包括隐藏文件
  • -l：列出所有文件，包括其详细信息

cd命令：切换目录

cd 目录名(绝对路径都是以/开头的，相对路径，对于当前目录该如何寻找../..)

pwd：显示当前用户所在的目录

mkdir：创建一个目录

-p：创建层级递归目录

删除目录、文件

  • rmdir：仅能删除空的目录，加-p参数可以递归删除多个目录。
  • rm -rf：强制删除。
  • rm -m：递归删除目录。
  • rm -i：互动，删除询问是否删除。

复制文件或目录

  • cp 原来的地方 新的地方

若要移动文件夹，加参数-r

移动文件、重命名文件

  • mv -f：强制
  • mu -u：只替换已经更新过的文件

更改文件属性

  • chmod 777 最高权限，所有人能访问
  • r：4，w：2，x：1

文件内容查看

Linux系统中使用一下命令来查看文件的内容：

  • cat 由第一行开始显示文件内容。
  • tac 从最后一行开始显示，是cat的倒写。
  • nl 显示的时候，顺道输出行号。
  • more 一页一页的显示文件内容，空格代表翻页，回车代表向下看一行
  • less与more类似，但是比more更好的是，他可以往前翻页（空格翻页，上下键代表翻动页面，退出为q命令，查找字符串/要查询的字符）。
  • head 只看头几行，通过-n参数来控制显示几行。
  • tail 只看尾巴几行，-n参数代表查看几行。

可以使用man来查看各个命令的使用文档。

Linux链接的概念

Linux的链接分为两种：硬链接、软链接。

硬链接：A—B，假设B是A的硬链接，A和B指向的同一个文件，允许一个文件拥有多个路径。用户可以通过这种机制可以建立硬链接到一些重要文件上，防止误删。

软链接：就是类似Windows下的快捷方式，删除了源文件，快捷方式也访问不了了。

  • ln：创建硬链接
  • ln -s：创建软链接
  • touch：创建文件
  • echo：输入字符串

f1为初始文件，f2为硬链接，f3为软链接

删除f1之后，查看f2和f3的区别，f2还在，f3软链接失效了

Vim编辑器

三种使用模式

命令模式，输入模式，底线命令模式

命令模式：

  • i切换到输入模式，输入字符
  • x删除当前光标所在处的字符
  • :切换到底线命令模式，在最底一行输入命令

底线命令模式：

  • q：退出程序
  • w：保存文件

我们一般把这两个一起用:wq，退出加保存。

如果存在这个文件，就打开这个文件，若文件不存在，则新建一个文件。

若上次编辑此文件崩溃时，会出现交换文件.swap，此时加-r参数恢复文件并且删除.swap文件。

账号管理

添加用户

  • useradd
  • useradd -m:自动创建这个用户的主目录 /home/
  • useradd -m username -G gruopname：给用户分配组

本质：Linux中一切皆文件，这里添加的用户其实就是往某一个文件中写入用户的信息了。/etc/passwd

删除用户

  • userdel -r : 删除用户时将其目录一并删除。

修改用户

  • usermod 对应修改的内容 修改的用户名

要去/etc/passwd中查看，对应的目录名修改了。

切换用户

  1. 切换用户的命令为：su username
  2. 从普通用户切换到root用户，可以使用命令：sudo su
  3. 在终端输入exit或logout或使用快捷方式ctrl+d，可以退回到原来用户
  4. 在切换用户时，如果想在切换用户之后使用新用户的工作环境，可以在su和username之间加-，例如su-root

$ 表示普通用户，# 表示超级用户，也就是root用户。

密码设置

我们一般通过root创建用户的时候，要配置密码。

如果是root：

  • passwd username

如果是普通用户

  • passwd

锁定账户

如果一个人辞职了，就可以冻结账号，这个人就登陆不上了。

  • passwd -l junge
  • passwd -d junge：这个用户没有密码，也不能登陆

用户组管理

每个用户都有一个用户组，系统可以对一个用户组中的所有用户进行集中管理。

创建一个用户组

  • groupadd junge

创建完用户组后可以得到一个组id，这个id是可以指定的。-g 1006 指定id，如果不指定则自增1

删除用户组

  • groupdel junge

修改用户组的权限信息和名字

  • groupmod -g 1002 junge：修改id
  • groupmod -n jungege junge：修改junge组名为jungege

文件的查看

/etc/passwd:

用户名:口令:用户标识号:组标识号:注释性描述:主目录:登录Shell

这个文件中每一行都代表一个用户，我们可以看到这个用户的主目录在哪里，可以看到属于哪个组。

真正的密码放在/etc/shadow中，密码都是加密的，保证密码的安全性。

磁盘管理

列出文件系统整体的磁盘使用量

  • df
  • df -h: 显示的是M

检查磁盘空间使用量

  • du

Mac或者Linux挂载本地磁盘或者文件

  • mount /dev/liljungege /mnt/liljungege
  • 将外部设备liljungege挂载到mnt目录下，来实现访问
  • 卸载：umount -f [名字] 强制卸载

如果安装了jdk，那可以用java中的命令来查看

进程管理

Linux中一切皆文件（文件：读写执行，权限：（用户，用户组），系统：（磁盘，进程））。

基本概念

  1. 在Linux中，每一个程序都有自己的一个进程，每一个进程都有一个ID号
  2. 每一个进程，都有一个父进程。
  3. 进程有两种存在方式：前台、后台。
  4. 一般的话服务都是后台运行的，基本的程序都是前台运行的。

命令

  • ps：查看当前系统中正在执行的各种进程的信息
  • ps -a: 显示当前终端运行的所有的进程信息
  • ps -u: 以用户的信息显示进程
  • ps -x：显示后台进程运行的参数
  • ps -aux|grep mysql
    • |在Linux中为管道符，A|B可以用操作A的结果输出去操作B
    • grep查找文件中符合条件的字符串
  • ps -ef：可以查看到父进程的信息
    • ps -ef|grep mysql

看父进程我们一般可以通过目录树结构来查看

  • pstree -pu
    • -p：显示父id
    • -u：显示用户组

  • kill -9 进程id: 结束进程
    • 平时如果java代码死循环了，就可以强制杀死进程
  • nohop: 将Java程序打包
